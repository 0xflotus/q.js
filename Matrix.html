Tables of numbers

A matrix is just a table of numbers; rows and columns of values. Matrices can be of any size. Here’s an example of a 3 × 2 matrix. It is 3 columns wide and 2 rows tall, containing the values 1 through 6. 


┌         ┐
│ 1  2  3 │
│ 4  5  6 │
└         ┘

When describing the dimensions of a matrix we always specify the number of rows first, then the number of columns. The above is an example of a 3 × 2 matrix, while below is a matrix containing similar data, but in a 2 × 3 configuration. 


┌      ┐
│ 1  2 │
│ 3  4 │
│ 5  6 │
└      ┘

Q thinks about matrices in row-major order. This means we read the values just as they are ordered above, starting with the top-most row, reading values from left to right, then proceeding to the next row down and repeating that process. Our choice of row-major order makes reading and writing matrix values more akin to reading and writing in English; easier to type and program here. To illustrate the converse, here’s an example of a 3 × 2 matrix in column-major order. 


┌         ┐
│ 1  3  5 │
│ 2  4  6 │
└         ┘

Constructor

Q.Matrix creates and operates on matrices of arbitrary dimensions. This is in contrast to many graphics libraries, for example, which optimize for specific sizes of matrices—with corresponding class names like Matrix3 (for 3 × 3 matrices) or Matrix4 (for 4 × 4 matrices). Q, meanwhile, is much more flexible. The constructor expects an argument list of equal-length arrays where each array represents a row of column values. Q.Matrix then automatically determines the dimensions based on the number of arguments (rows) and length of each row (number of columns). The constructor will throw an error if the row lengths are not equal.

Upon creation Q.Matrix converts all argument values of type Number to Q.ComplexNumber instances. Currently Q.Matrix does not support recursion; a matrix cannot contain matrices as values. 


var a = new Q.Matrix(  

	[ 1, 2, 3 ],  
	[ 4, 5, 6 ]
)

Import and export formats

Getting data in and out of any system can be a laborious. With that in mind, Q.Matrix comes with helper methods for importing and exporting the following common formats:

  • Comma separated values (CSV)
  • Tab separated values (TSV)
  • Arbitrary character separated values (XSV)
  • HTML tables
