Operations, not just matrices

Traditionally, a quantum logic gate is a matrix that can be multiplied against a qubit’s matrix. For our purposes we need a broad definition. Let’s consider a Gate to be an operation. That operation can be the multiplication of a logic gate matrix against a qubit matrix. Or it can be a measurement operation that collapses a qubit. It can also be a visualization operation, such as a Bloch Sphere visualization of a qubit state. When gates are attached to a Board they are placed in both space and time.

Constructor

Expects a function as a first argument that will become the operation invoked by .applyTo(). Accepts a string as an optional second argument (expected to be a single character) for use as a label. Accepts a string as an optional third argument for use as a name / longer label. 


var g = new Q.Gate( 

	function( input ){

		console.log( 'You input this:', input )
	},
	'?', 'Input mystery!'
)
  • label
    String. A single character used as a label in visual representations of this Gate in circuit diagrams.
  • name
    String. The full name of this Gate to be used as a title or long label in a visual representation.
  • applyTo
    Function. The main operation of this Gate.
  • index
    Number. An auto-incrementing identification number assigned to the instance, used for minding the total number of instances created.

Constants

While Q.Gate constants are appended directly to the Q.Gate object, they are also appended to a Q.Gate.constants object to make looking up constants in the JavaScript console trivial, and to make iterating across all constants convenient via functions like Object.entries, Object.keys, Object.values, and so on.

  • IDENTITY
    
new Q.Gate( function( qubit ){ 

		return qubit 
	},
	'I', 'Identity' )
  • MEASURE
    
new Q.Gate( function( qubit ){ 

		return qubit.collapse() 
	},
	'M', 'Measure' )
