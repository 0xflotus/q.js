Imaginary numbers: 𝕀

Now let’s consider the imaginary number named 𝒊. What makes 𝒊 imaginary? And why is an imaginary number even useful? Recall that any number multiplied by itself will always be a positive number. For example, 7 × 7 = 49. But also -7 × -7 = 49. For brevity we could rewrite those equations as 72 = 49 and -72 = 49, respectively. No matter what real number you try, when you multiply it by itself the product will be a positive number. Try solving for 𝒙 in this equation: 𝒙² = 49. You’ll find that 𝒙 can equal 7 and 𝒙 can also equal -7. Both are true. But now try solving for 𝒙 in this equation: 𝒙² = -49. There is no real solution.

Here comes 𝒊 to the rescue. The imaginary number 𝒊 represents the impossible solution to the square root of negative one. We can express that elegantly in two ways: 


𝒊² = −1

𝒊 = √-1

This is what Monty Python’s John Cleese might call the impossible intermediate. And with it we can now sort of solve for 𝒙 in 𝒙² = -49: 


𝒙² = -49

𝒙  = √-49

𝒙  = √(49 × -1)

𝒙  = √49 × √-1

𝒙  = 7 × √-1

𝒙  = 7 × 𝒊

𝒙  = 7𝒊

Our result, 7𝒊, is an imaginary number that consists of a real number multiplier, 7, and our special imaginary number 𝒊. Imaginary numbers also exist along an infinite number line—with zero in the middle, negative numbers to the left, and positive to the right:

Complex numbers: ℂ

We just saw that multiplying a real number (ℝ) by the imaginary number 𝒊 yields a number that is also imaginary (𝕀). But what if you add a real number to an imaginary one? Things get complex. A complex number is a number that can be expressed in the form 𝒂 + 𝒃𝒊, where 𝒂 is the real component (ℝ) and 𝒃𝒊 is the imaginary component (𝕀). Some examples might be 1 + 2𝒊 or 3 - 4𝒊. This is what the ComplexNumber class was built to handle. Open up your JavaScript console and paste in the following: 


var 
a = new Q.ComplexNumber( 1,  2 ),
b = new Q.ComplexNumber( 3, -4 )

Now we have two variables, 𝒂 and 𝒃, that we can operate with. As you might guess, ComplexNumber includes instance methods for common operations like addition, subtraction, multiplication, and divion. It also includes a handy toString for expressing a complex number in our familiar way. For example a.toString() will output 1 + 2i. Try the following lines individually in your JavaScript console: 


'𝒂 = '+ a.toString()
'𝒃 = '+ b.toString()

'𝒂 + 𝒃 = '+ a.add( b ).toString()
'𝒂 - 𝒃 = '+ a.subtract( b ).toString()
'𝒂 × 𝒃 = '+ a.multiply( b ).toString()
'𝒂 ÷ 𝒃 = '+ a.divide( b ).toString()

Operation functions on Q.ComplexNumber instances generally accept as arguments both sibling instances and pure Number instances, though the value returned is always an instance of Q.ComplexNumber.
